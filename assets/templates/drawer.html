基本设置
关闭后顶部图标完全隐藏，不留空位。可通过扩展面板（积木图标）中的「Horae 时光记忆」栏位重新开启。
消息面板占聊天区域的宽度百分比（50-100）。宽屏美化可调低避免溢出。
外观设置
选择内置主题或导入的自定义美化。导入美化文件（.json）后会出现在下拉列表中。
自定义CSS
输入自定义CSS覆盖插件样式。适合进阶用户微调配色。修改后切换焦点即生效。
高级设置
控制发送给AI的"一般"级别剧情轨迹数量：
0 = 不发送一般事件（重要/关键照发）
15 = 发送最近15条一般事件（默认）
数字越大发送越多，Token消耗越高
Horae状态信息插入到AI提示词的哪个位置：
0 = 最末尾（紧贴最新消息）
1 = 倒数第二（推荐，给AI留思考空间）
数字越大越靠前
自定义表格
创建Excel风格表格，让AI按你的需求填写特定信息。
长按表头可添加/删除行列。每个表格可设置提示词指导AI如何填写。
本地表格跟随当前对话保存；全局表格所有对话共享。点击表格名称前的图标可切换。
🔒 锁定行列可防止AI修改特定内容。
发送给AI的内容
控制哪些信息会随提示词发送给AI。关闭可节省Token，但AI将无法"记住"这些内容。
当前注入 Token：计算中...
历史事件摘要、相对时间计算。关闭后AI无法知道"几天前"发生了什么。
在场角色、服装描述、好感度、角色信息。关闭后AI可能描写错误的服装。
当前持有的物品及位置。关闭后AI可能"凭空"取出不存在的物品。
记录地点的固定物理特征，保持跨回合场景描写一致性。开启后AI会在首次到达新地点时记录特征描述。
追踪角色间的关系变化（朋友、恋人、敌对等），AI仅在关系变化时输出。关闭后不生成rel:标签，不注入关系网络提示词。
追踪在场角色的情绪/心理状态变化，帮助AI保持角色情绪连贯性。AI仅在情绪明显变化时输出。
自动摘要 & 隐藏
超过缓冲上限后自动将旧事件交给AI压缩为摘要，节省Token。
压缩后的摘要可在时间线中切换回原始事件查看。
最近N条消息的事件不会被自动压缩，保留近期剧情细节。
超过此上限时触发自动压缩。
每次自动摘要最多处理多少条消息。旧档案首次启用时会分多轮逐步消化。
每次摘要送入的文本 Token 上限，超出则截断至此值。
参考各模型上下文窗口（建议设为 30%~50%）：
Claude ≈ 80K~200K · GPT-4o ≈ 80K~128K · Gemini ≈ 100K~1000K · DeepSeek ≈ 64K
填入一个 OpenAI 兼容端点，摘要生成将直接请求该端点，不占用酒馆主连接。
OpenAI 兼容端点基础地址，如 https://api.openai.com/v1
低成本模型即可，如 gpt-4o-mini、deepseek-chat 等。
填写完成后，自动摘要将通过独立 HTTP 请求并行生成，不影响酒馆主 API。
支持所有 OpenAI 兼容格式的端点（OpenAI、DeepSeek、OpenRouter 等）。
