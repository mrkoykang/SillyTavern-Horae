自动摘要 & 隐藏
超过缓冲上限后自动将旧事件交给AI压缩为摘要，节省Token。
压缩后的摘要可在时间线中切换回原始事件查看。
最近N条消息绝对不会被摘要。确保近期剧情细节完整保留。
超过此上限时触发自动压缩。
举例：假设保留=10，缓冲上限=20（楼层模式）
→ 你聊到第 50 楼时：第 41-50 楼绝对保留不摘要
→ 第 1-40 楼中，未被摘要的部分如果 超过 20 楼就触发一次自动摘要
→ 每次摘要按照下方设置的批次上限（消息条数或Token数）处理一批，剩余的留到下一轮
简单理解：保留 + 缓冲 = 多少楼之后才开始摘要
每次自动摘要最多处理多少条消息。旧档案首次启用时会分多轮逐步消化。
每次摘要送入的文本 Token 上限，超出则截断至此值。
参考各模型上下文窗口（建议设为 30%~50%）：
Claude ≈ 80K~200K · GPT-4o ≈ 80K~128K · Gemini ≈ 100K~1000K · DeepSeek ≈ 64K
填入一个 OpenAI 兼容端点，摘要生成将直接请求该端点，不占用酒馆主连接。
OpenAI 兼容端点基础地址，如 https://api.openai.com/v1
填好地址和密钥后点击 拉取。低成本模型即可。
填写完成后，自动摘要将通过独立 HTTP 请求并行生成，不影响酒馆主 API。
支持所有 OpenAI 兼容格式的端点（OpenAI、DeepSeek、OpenRouter 等）。
连接测试仅查询模型列表，不消耗生成次数。
向量记忆 (Beta)
对 Horae 标签数据做向量检索，自动召回相关历史片段注入给 AI。
本地模式：下载约 25MB 量化模型到浏览器缓存，占用 50-80MB 内存，⚠ 2GB 以下手机或云酒馆不建议使用，请务必确保有配置—max-old-space-size解除Node.js运行内存限制再启用。
API 模式：使用远程 Embedding API，无需本地资源。
q4 适合大多数设备；q8 精度更高但体积翻倍。
只能填写 Embedding（嵌入/向量）模型！
不要填 LLM 聊天模型（如 gemini 3、gpt-4 等），否则会报错。
推荐：text-embedding-v3（Qwen）、text-embedding-3-small（OpenAI）
OpenAI 兼容格式，会自动拼接 /embeddings 路径。
例，填：https://dashscope.aliyuncs.com/compatible-mode/v1
填好地址和密钥后点击 拉取。
推荐: text-embedding-v3 (Qwen) / text-embedding-3-small (OpenAI)
适用于高维度 API 模型（如 Qwen3-Embedding、text-embedding-3-large 等）。
开启后将关闭关键词扩展、事件词组匹配、高频词惩罚等辅助启发式，
完全依赖模型自身的语义理解能力进行召回，减少低质量噪音。
如果你使用的是小型或低维度模型，请不要开启，否则召回质量会下降。
向量召回后，再用 Rerank 模型对候选结果做精细排序，显著提升召回精度，
把真正相关的排到前面，不太相关的压下去，让 AI 收到的记忆更准确。
需要额外配置一个 Rerank 模型的 API（和 Embedding 模型不同），
推荐模型：Qwen3-Reranker、bge-reranker-v2-m3、bce-reranker
填好地址和密钥后点击 拉取。
推荐: gte-rerank (Qwen) / jina-reranker-v2-base-multilingual (Jina)
默认 Rerank 使用摘要进行排序。开启后将使用原始消息全文进行精排，
可以捕捉到摘要遗漏的细节，但对模型的上下文长度要求很高。
仅适用于长上下文 Rerank 模型（如 Qwen3-Reranker 系列，支持 32K）。
bge-reranker-v2-m3（≤1024 有效）和 bce-reranker（≤512）等短上下文模型
请勿开启，全文会被截断，效果反而更差。
每次发送消息时，从向量库召回最相关的 K 条记忆注入上下文。
相似度低于此值的记忆不会被召回。提高此值可减少噪音。
排名前 N 且达到全文阈值的召回结果会发送原始正文而非摘要。设为 0 关闭此功能。
只有相似度达到此值的高置信度结果才会发送全文，避免不相关内容浪费 Token。
用逗号分隔标签名。以下场景会整段移除这些标签包裹的内容：
· 向量检索全文回顾 & Rerank 全文精排
· 时间线解析：防止小剧场内的 horae 标签污染正文时间线
· AI 智能摘要：发送消息给 AI 分析前先剔除
例：填 snow, theater 会剔除 <snow>...</snow> 和 <theater>...</theater>。
